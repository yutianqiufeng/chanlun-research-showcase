# Open / Private Boundary

## Open Scope

The public repository may include:

- data organization patterns
- case export schemas
- plotting helpers
- evaluation helpers
- public reports
- architecture and methodology notes

## Private Scope

The following remain private and are not published here:

- buy-point recognition logic
- filtering conditions and filter ordering
- multi-timeframe trigger details
- signal scoring and pruning logic
- execution thresholds
- live trading implementation details
- risk-control implementation details

## Design Rule

If a file explains why a signal was selected or rejected, it should stay private.

If a file only verifies already-exported public cases, it can be public.
